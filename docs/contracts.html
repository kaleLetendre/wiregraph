Deep dive

A contract is a defined communication between two compartments.

Not just microservices. Any two compartments — two services, a library and its consumer, two programs sharing a table — agree on a shape and talk through it. That agreed shape is the contract. It's the seam a call graph never records, because no function ever calls across it.

The boundary is the point. Inside a compartment, everything resolves — you can follow calls symbol to symbol. Cross the boundary and there is no call to follow: A doesn't invoke B, it just conforms to the same shape B expects. The contract is that shape, made into a node so the crossing becomes traceable.

The variety

Same idea, three surfaces.

A contract isn't only an HTTP call between services. It's any shape two compartments agree on — over a wire, through an exported API, or in shared state.

service ↔ service

The wire

An HTTP route or a queue topic. One service serves or publishes it; another calls or consumes it.

library ↔ consumer

The exported API

A package exposes a surface — functions, types, an SDK. Another repo imports it and depends on those names.

program ↔ program

Shared state

An env var, a database table, a file on disk. Two programs never speak — they meet at a shared key.

The invisible bug class

Rename one side. The other breaks — silently.

Because no code calls across the seam, nothing in either repo references the other. Your compiler is happy. Your tests in each repo pass. The break only surfaces at runtime, in production, between the two.

✕ grep for /api/register in repo A — clean ✕ a call graph — no edge ever crossed the repo ✓ wiregraph — the contract's two edges no longer agree

How wiregraph detects it

Four detectors. One rule.

Each detector pulls a candidate token out of the code. When the same token shows up in 2+ repos, wiregraph mints a Contract node and links each side to it.

wire

HTTP routes

A route one repo serves and another calls.

/api/register
→ REFERENCES a Contract
message

Queues / topics

A topic a producer emits and a consumer handles.

orders.created
→ REFERENCES a Contract
state

Shared state

An env var or key two services both touch.

DATABASE_URL
→ REFERENCES a Contract
import

Library / SDK

A package surface one repo exposes, another imports.

import @acme/sdk
→ IMPORTS the package
⚑
Detection is a heuristic, and contracts are proposed, not written. wiregraph reads the shared tokens straight out of your code and drafts the seam — it never edits your source or invents a spec you didn't declare. Run /wiregraph-contracts to review the inferred wires as a draft AsyncAPI spec before you trust them.
