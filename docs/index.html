The differentiator

Works with your contract architecture.

Within a repo, calls resolve by name. Across repos there's usually no call at all — code communicates through a shared shape. wiregraph detects that shape and draws the seam a call graph can't see.

  • →
    A route, a queue topic, an env var, or an imported package — when a token appears in 2+ repos, wiregraph mints a Contract node.
  • →
    Producer and consumer both link to it with REFERENCES edges, so Claude can trace producer → contract → consumer across the boundary.
  • →
    Rename one side and the seam is right there — no grep, no guessing which of your services just broke.

Install

Three lines. No build step.

Public repo, install straight from Claude Code. Native bits are prebuilt and vendored for Linux / macOS / Windows.

claude-code
/plugin marketplace add kaleLetendre/wiregraph
/plugin install wiregraph@wiregraph
/reload-plugins
/wiregraph-init
# → indexes every repo under this workspace, then keeps itself fresh

Run /wiregraph-init once per workspace. It finds every repo under the root, indexes them into one graph, and re-indexes as you edit. Then just put Claude to work.
