How it works

Source in. A queryable graph out.

tree-sitter parses your files into symbols, calls, and candidates. wiregraph resolves them, links contracts, and stores it all in one local SQLite file that Claude reads through MCP tools.

The differentiator

Links code across repos — through contracts.

Two repos never call each other in code. They share only a token — a route, a topic, an env var, an import. The instant they do, wiregraph draws the seam.

Install

Three lines. No build step.

Public repo, install straight from Claude Code. Native bits are prebuilt and vendored for Linux / macOS / Windows.

claude-code
/plugin marketplace add kaleLetendre/wiregraph
/plugin install wiregraph@wiregraph
/reload-plugins
/wiregraph-init
# → indexes every repo under this workspace, then keeps itself fresh

Run /wiregraph-init once per workspace. It finds every repo under the root, indexes them into one graph, and re-indexes as you edit. Then just put Claude to work.
