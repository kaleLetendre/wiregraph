The problem

grep reads everything. wiregraph reads what matters.

Ask an agent "who calls this?" and it greps the whole tree, then reads whole files to be sure. You pay for every token. wiregraph answers from the graph — precise, and about half the cost.

How it works

One pass. Source in, queryable graph out.

Walk each repo, parse with tree-sitter, resolve calls in-repo, link the seams through contracts, and load it all into one embedded SQLite file — served to Claude as MCP tools.

The differentiator

Two repos. No call between them. Linked anyway.

The most valuable connections aren't calls — they're contracts: the agreed shape at the seam between two compartments. When a distinctive token shows up in 2+ repos, wiregraph joins them producer → consumer.

A distinctive token shared by ≥ 2 repos becomes a seam — the link no grep and no call graph would ever find.

Every kind of seam, one machinery

Four detectors harvest cross-repo signals. A distinctive token shared by 2+ repos becomes a link.

wire

HTTP routes — Express, FastAPI, fetch, axios. Token = the path.

message

Queues & topics — Kafka, AMQP, MQTT, NATS. Token = the topic.

state

Shared env vars — process.env, getenv. Token = the var name.

import

Cross-repo deps — import / require / #include. A direct IMPORTS edge.

Why it matters: the otherwise-invisible bug

Rename a field on the producing side and the consumer silently reads the old one. grep can't see across repos; a call graph has no edge to follow. A contract graph makes that cross-boundary blast radius visible — before it ships broken.

What Claude can do

Question-shaped tools. You don't call them — Claude does.

The graph is exposed as MCP tools. Claude reaches for the right one, gets a precise answer, and reads only the source that matters.

find_symbolanswers where something is defined.
get_sourceanswers with one symbol's body — not the whole file.
trace_callersanswers who calls it — the whole tree, one query.
trace_calleesanswers what it calls — the whole tree, one query.
trace_contractanswers who's on each side of a seam, across repos.
path_betweenanswers how two symbols connect — calls and contracts.
query_sqlanswers anything else — read-only SQL over the graph.
/wiregraph-statsanswers how many tokens the graph has saved you.
Install

Three commands, then index once.

Add the plugin, reload, and run init in a folder of side-by-side repos. Everything stays local.

1 — Add & install the plugin
claude code
›/plugin marketplace add kaleLetendre/wiregraph
›/plugin install wiregraph@wiregraph
›/reload-plugins
2 — Index a workspace of related repos
side-by-side repos under one folder
›/wiregraph-init # lists repos, confirms scope, builds the graph

That's it. Claude now works from the full picture — every caller, the real blast radius, cross-repo wiring — at roughly half the tokens.
